# Principles

## Quality Bar

Work must be small, reviewed, and backed by focused evidence.

## TDD And Regression Policy

Implementation starts with a failing test, fixture eval, or scenario eval.

## Review Rules

Review checks behavior, evidence, and unintended changes before completion.

## Security And Safety

Avoid unsafe writes and keep runtime data local.

## Non-Negotiables

Deterministic commands own `.hub` mutations.

## Tradeoffs

Prefer stable contracts over broad automation.
